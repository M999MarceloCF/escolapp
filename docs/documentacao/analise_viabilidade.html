Análise de Viabilidade - EscolApp

Introdução

Este documento apresenta a análise de viabilidade do EscolApp, um sistema simplificado de gestão escolar para dispositivos móveis, desenvolvido para atender às necessidades da Escola Municipal Professor Paulo Freire.

Viabilidade Técnica

A análise de viabilidade técnica avalia se o projeto é tecnicamente possível, considerando as tecnologias disponíveis e as restrições do ambiente.

Tecnologias Propostas

  • Linguagem de Programação: Python
  • Framework Mobile: Kivy/KivyMD
  • Banco de Dados Local: SQLite
  • Padrão Arquitetural: MVC (Model-View-Controller)

Justificativa da Escolha Tecnológica

  • Python: Linguagem de fácil aprendizado e manutenção, com ampla comunidade e bibliotecas.
  • Kivy/KivyMD: Framework multiplataforma que permite desenvolvimento para Android com baixo consumo de recursos.
  • SQLite: Banco de dados leve, que não requer servidor, ideal para aplicações móveis com funcionamento offline.
  • MVC: Padrão que facilita a manutenção e evolução do código, separando a lógica de negócio da interface.

Requisitos de Hardware

O aplicativo foi projetado para funcionar em dispositivos com configurações modestas:

  • Android 6.0 ou superior
  • 1GB de RAM (mínimo)
  • 100MB de espaço de armazenamento disponível
  • Processador de 1.2GHz (mínimo)

Conclusão da Viabilidade Técnica

O projeto é tecnicamente viável, utilizando tecnologias estabelecidas e de código aberto, com requisitos de hardware acessíveis mesmo para dispositivos mais antigos ou de entrada.

Viabilidade Econômica

A análise de viabilidade econômica avalia os custos e benefícios do projeto.

Custos Estimados

Item Custo Estimado
Desenvolvimento do Software Baixo (utilizando tecnologias open source)
Infraestrutura de Servidor Mínimo (sincronização ocasional)
Manutenção Baixo (código simples e bem documentado)
Treinamento Médio (interface intuitiva reduz necessidade)

Benefícios Esperados

  • Redução do tempo gasto em tarefas administrativas
  • Melhoria na comunicação entre escola e famílias
  • Redução de custos com papel e impressão
  • Acesso mais rápido e fácil às informações escolares
  • Maior engajamento dos responsáveis no acompanhamento escolar

Conclusão da Viabilidade Econômica

O projeto apresenta boa viabilidade econômica, com custos reduzidos devido ao uso de tecnologias open source e benefícios significativos para a gestão escolar e comunicação.

Viabilidade Operacional

A análise de viabilidade operacional avalia se o sistema será utilizado efetivamente após sua implementação.

Fatores Favoráveis

  • Interface simples e intuitiva, adequada para usuários com diferentes níveis de familiaridade com tecnologia
  • Funcionamento offline, permitindo uso mesmo com conexão limitada
  • Baixo consumo de recursos, compatível com dispositivos mais antigos
  • Atende a necessidades reais e urgentes da escola

Desafios Operacionais

  • Necessidade de treinamento inicial para professores e funcionários
  • Possível resistência à mudança por parte de alguns usuários
  • Necessidade de suporte técnico para resolução de problemas

Estratégias para Superar Desafios

  • Realizar treinamentos práticos e fornecer material de apoio
  • Implementar o sistema gradualmente, começando com funcionalidades básicas
  • Designar "multiplicadores" entre os usuários para auxiliar colegas
  • Fornecer canal de suporte técnico acessível

Conclusão da Viabilidade Operacional

O projeto apresenta boa viabilidade operacional, com interface intuitiva e funcionamento adaptado à realidade da escola pública. Os desafios identificados podem ser superados com estratégias adequadas de implementação e treinamento.

Conclusão Geral

A análise de viabilidade indica que o EscolApp é um projeto viável do ponto de vista técnico, econômico e operacional. O sistema proposto atende às necessidades da Escola Municipal Professor Paulo Freire, utilizando tecnologias acessíveis e adequadas ao contexto de uma escola pública com recursos limitados.

Recomenda-se a continuidade do projeto, com atenção especial ao treinamento dos usuários e à implementação gradual das funcionalidades.
