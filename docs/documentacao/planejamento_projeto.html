Planejamento do Projeto - EscolApp

Introdução

Este documento apresenta o planejamento do projeto EscolApp, um sistema simplificado de gestão escolar para dispositivos móveis, desenvolvido para atender às necessidades da Escola Municipal Professor Paulo Freire.

Metodologia de Desenvolvimento

O projeto será desenvolvido utilizando uma metodologia ágil adaptada, com ciclos de desenvolvimento curtos e entregas incrementais. Esta abordagem foi escolhida por:

  • Permitir feedback constante do cliente
  • Facilitar adaptações e ajustes durante o desenvolvimento
  • Possibilitar entregas parciais de funcionalidades
  • Ser adequada para equipes pequenas

Equipe do Projeto

A equipe do projeto é composta por:

Função Responsabilidades
Gerente de Projeto Coordenação geral, comunicação com o cliente, gestão de recursos
Analista de Requisitos Levantamento e documentação de requisitos, validação com o cliente
Desenvolvedor Frontend Desenvolvimento da interface do aplicativo, experiência do usuário
Desenvolvedor Backend Desenvolvimento da lógica de negócio, banco de dados, sincronização
Testador Planejamento e execução de testes, validação de qualidade

Cronograma

O projeto será desenvolvido em um período de 12 semanas, dividido nas seguintes fases:

Fase Duração Atividades Principais
Iniciação 1 semana Definição do escopo, análise inicial, formação da equipe
Planejamento 2 semanas Levantamento detalhado de requisitos, definição de arquitetura, planejamento de sprints
Execução - Sprint 1 2 semanas Desenvolvimento do módulo de autenticação e perfis de usuário
Execução - Sprint 2 2 semanas Desenvolvimento do módulo de registro de notas e frequência
Execução - Sprint 3 2 semanas Desenvolvimento do módulo de comunicados e calendário
Execução - Sprint 4 2 semanas Desenvolvimento do módulo de ocorrências e sincronização
Encerramento 1 semana Testes finais, documentação, treinamento, entrega

Estimativas de Recursos

Os recursos necessários para o desenvolvimento do projeto incluem:

Recursos Humanos

  • 1 Gerente de Projeto (dedicação parcial)
  • 1 Analista de Requisitos (dedicação integral nas fases iniciais)
  • 1 Desenvolvedor Frontend (dedicação integral)
  • 1 Desenvolvedor Backend (dedicação integral)
  • 1 Testador (dedicação parcial)

Recursos Tecnológicos

  • Computadores para desenvolvimento
  • Dispositivos Android para testes
  • Ambiente de desenvolvimento Python/Kivy
  • Ferramentas de controle de versão (Git)
  • Ferramentas de gestão de projeto

Gestão de Riscos

Os principais riscos identificados para o projeto são:

Risco Probabilidade Impacto Estratégia de Mitigação
Mudanças nos requisitos Média Alto Documentação clara, validações frequentes com o cliente
Dificuldades técnicas com Kivy Média Médio Capacitação prévia, protótipos iniciais, plano B com tecnologia alternativa
Atrasos no cronograma Alta Médio Monitoramento constante, priorização de funcionalidades, buffer no cronograma
Problemas de sincronização offline/online Alta Alto Testes extensivos, implementação gradual, foco inicial no modo offline
Resistência dos usuários Média Alto Envolvimento dos usuários desde o início, treinamento adequado, interface intuitiva

Plano de Comunicação

A comunicação no projeto será gerenciada da seguinte forma:

  • Reuniões diárias: Breves encontros da equipe para alinhamento
  • Reuniões de sprint: Planejamento e revisão a cada duas semanas
  • Reuniões com o cliente: Quinzenais para validação e feedback
  • Ferramentas de comunicação: E-mail, mensageria instantânea, sistema de gestão de projetos
  • Documentação: Compartilhada em repositório acessível a todos os envolvidos

Critérios de Aceitação

O projeto será considerado concluído com sucesso quando:

  • Todas as funcionalidades essenciais estiverem implementadas e testadas
  • O aplicativo funcionar adequadamente em modo offline
  • A sincronização de dados funcionar corretamente
  • Os testes de usabilidade com usuários reais forem satisfatórios
  • A documentação estiver completa
  • O cliente formal aprovar a entrega
