Protótipos de Interface - EscolApp

Introdução

Este documento apresenta os protótipos de interface do EscolApp, um sistema simplificado de gestão escolar para dispositivos móveis, desenvolvido para atender às necessidades da Escola Municipal Professor Paulo Freire.

Os protótipos foram desenvolvidos seguindo princípios de design centrado no usuário, com foco em simplicidade, usabilidade e acessibilidade, considerando as limitações de dispositivos móveis mais antigos e o contexto de uma escola pública.

Diretrizes de Design

Os protótipos seguem as seguintes diretrizes de design:

  • Simplicidade: Interface limpa e direta, sem elementos desnecessários
  • Consistência: Padrões visuais e de interação consistentes em toda a aplicação
  • Acessibilidade: Textos legíveis, contraste adequado, elementos de tamanho suficiente para toque
  • Eficiência: Fluxos de trabalho otimizados, minimizando o número de toques necessários
  • Feedback: Respostas claras às ações do usuário
  • Desempenho: Elementos visuais leves para garantir bom desempenho em dispositivos mais antigos

Paleta de Cores

A paleta de cores foi escolhida para transmitir confiança, seriedade e um ambiente educacional, além de garantir boa legibilidade:

  • Cor primária: Azul (#1976D2) - Utilizada em cabeçalhos, botões principais e elementos de destaque
  • Cor secundária: Amarelo (#FFC107) - Utilizada em elementos de destaque secundários e acentos
  • Cor de fundo: Branco (#FFFFFF) para áreas de conteúdo, Cinza claro (#F5F5F5) para fundo geral
  • Cor de texto: Cinza escuro (#333333) para texto principal, Cinza médio (#666666) para texto secundário
  • Cores de feedback: Verde (#4CAF50) para sucesso, Vermelho (#F44336) para erro, Amarelo (#FFC107) para alerta

Tipografia

A tipografia foi escolhida para garantir boa legibilidade em diferentes tamanhos de tela:

  • Fonte principal: Roboto - Uma fonte sem serifa, clara e moderna
  • Tamanhos: 16px para texto principal, 14px para texto secundário, 20px para títulos, 24px para cabeçalhos principais
  • Pesos: Regular para texto normal, Bold para títulos e elementos de destaque

Descrição

A tela de login é a porta de entrada do aplicativo. Ela foi projetada para ser simples e direta, solicitando apenas as informações essenciais (usuário e senha).

Elementos principais

  • Logo e nome do aplicativo
  • Campos de entrada para usuário e senha
  • Link para recuperação de senha
  • Botão de login
  • Informações da escola e versão do aplicativo

Considerações de usabilidade

  • Campos de entrada grandes para facilitar a digitação em dispositivos móveis
  • Botão de login amplo e destacado
  • Opção de recuperação de senha facilmente acessível
  • Feedback visual claro para erros de autenticação (não mostrado no protótipo)

Descrição

O menu principal é o hub central do aplicativo, oferecendo acesso a todas as funcionalidades disponíveis para o perfil do usuário. Este exemplo mostra o menu para um usuário com perfil de Professor.

Elementos principais

  • Cabeçalho com identificação do usuário e seu perfil
  • Lista de funcionalidades disponíveis
  • Indicadores visuais para notificações não lidas
  • Botão de logout

Considerações de usabilidade

  • Itens de menu amplos para facilitar o toque
  • Descrições claras para cada funcionalidade
  • Indicadores visuais destacados para chamar atenção para itens que requerem ação
  • Organização das funcionalidades por ordem de importância e frequência de uso

Adaptação por perfil

O menu principal se adapta automaticamente ao perfil do usuário:

  • Administrador: Acesso a todas as funcionalidades, incluindo gerenciamento de usuários
  • Professor: Acesso a registro de notas, frequência, comunicados, calendário e ocorrências
  • Aluno: Acesso a visualização de notas, frequência, comunicados e calendário
  • Responsável: Acesso a visualização de notas, frequência, comunicados, calendário e ocorrências dos alunos sob sua responsabilidade

Descrição

A tela de registro de notas permite aos professores inserir e editar as notas dos alunos de uma turma específica.

Elementos principais

  • Informações da turma, disciplina e bimestre
  • Lista de alunos com campos para inserção de notas
  • Botões para salvar ou cancelar as alterações
  • Informações sobre a última atualização e status de sincronização

Considerações de usabilidade

  • Layout tabular para facilitar a visualização e edição de múltiplas notas
  • Campos de entrada numérica otimizados para digitação de notas
  • Feedback claro sobre o status de sincronização
  • Opção de cancelar para evitar alterações acidentais

Funcionalidades adicionais

  • Validação de entrada para garantir que as notas estejam dentro da escala permitida (0-10)
  • Salvamento automático local para evitar perda de dados
  • Sincronização quando houver conexão com a internet
  • Histórico de alterações para rastreabilidade

Descrição

A tela de comunicados permite visualizar mensagens enviadas pela escola, professores e outros usuários, além de permitir o envio de novos comunicados (dependendo do perfil do usuário).

Elementos principais

  • Botão para criar novo comunicado (visível apenas para perfis autorizados)
  • Lista de comunicados recebidos, ordenados por data (mais recentes primeiro)
  • Informações sobre remetente, data e público-alvo de cada comunicado
  • Indicadores visuais de status de leitura
  • Opção para carregar mais comunicados

Considerações de usabilidade

  • Destaque visual para comunicados não lidos
  • Informações essenciais visíveis sem necessidade de abrir o comunicado
  • Categorização visual por tipo de comunicado (geral, específico para turma)
  • Carregamento progressivo para otimizar desempenho

Funcionalidades adicionais

  • Filtros por tipo de comunicado, remetente ou data
  • Busca por palavras-chave
  • Notificações para novos comunicados
  • Opção de responder a comunicados (quando aplicável)

Considerações Finais

Os protótipos apresentados representam as principais telas do EscolApp, demonstrando a abordagem de design centrado no usuário e focado na simplicidade e usabilidade.

Todas as telas foram projetadas considerando:

  • O contexto de uso em uma escola pública
  • As limitações de dispositivos móveis mais antigos
  • A possibilidade de uso offline
  • A diversidade de usuários, com diferentes níveis de familiaridade com tecnologia

Os protótipos servirão como base para o desenvolvimento da interface do aplicativo, podendo sofrer ajustes durante a implementação para atender a requisitos técnicos específicos ou feedback dos usuários.
