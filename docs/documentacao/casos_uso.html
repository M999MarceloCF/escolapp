Casos de Uso - EscolApp

Diagrama de Casos de Uso

O diagrama de casos de uso do EscolApp representa as principais funcionalidades do sistema e como os diferentes tipos de usuários (atores) interagem com essas funcionalidades.

Atores do Sistema

  • Administrador: Responsável pela gestão do sistema, cadastro de usuários e configurações gerais.
  • Professor: Registra notas, frequência e ocorrências, além de enviar comunicados.
  • Aluno: Visualiza notas, frequência, comunicados e calendário escolar.
  • Responsável: Visualiza informações dos alunos sob sua responsabilidade e comunica-se com a escola.

Casos de Uso Detalhados

UC01 - Autenticar Usuário

Ator Principal: Todos os usuários

Descrição: Este caso de uso permite que os usuários acessem o sistema através de credenciais de login.

Fluxo Principal:

  1. Usuário inicia o aplicativo
  2. Sistema apresenta tela de login
  3. Usuário insere nome de usuário e senha
  4. Sistema valida as credenciais
  5. Sistema direciona o usuário para a tela principal correspondente ao seu perfil

Fluxo Alternativo: Credenciais inválidas - Sistema exibe mensagem de erro e solicita nova tentativa

UC02 - Gerenciar Usuários

Ator Principal: Administrador

Descrição: Este caso de uso permite ao administrador cadastrar, editar e desativar usuários no sistema.

Fluxo Principal:

  1. Administrador acessa a opção "Gerenciar Usuários"
  2. Sistema exibe lista de usuários cadastrados
  3. Administrador seleciona a ação desejada (cadastrar, editar ou desativar)
  4. Sistema processa a ação e exibe mensagem de confirmação

UC03 - Registrar Notas e Frequência

Ator Principal: Professor

Descrição: Este caso de uso permite ao professor registrar notas e frequência dos alunos.

Fluxo Principal:

  1. Professor acessa a opção "Registro de Notas e Frequência"
  2. Sistema exibe lista de turmas do professor
  3. Professor seleciona a turma desejada
  4. Sistema exibe lista de alunos da turma
  5. Professor insere notas ou registra frequência
  6. Sistema armazena os dados localmente
  7. Sistema sincroniza os dados quando houver conexão com a internet

UC04 - Enviar Comunicados

Ator Principal: Administrador, Professor

Descrição: Este caso de uso permite enviar comunicados para alunos e responsáveis.

Fluxo Principal:

  1. Usuário acessa a opção "Comunicados"
  2. Usuário seleciona "Novo Comunicado"
  3. Usuário seleciona os destinatários (turmas ou usuários específicos)
  4. Usuário redige o comunicado e anexa arquivos (se necessário)
  5. Usuário envia o comunicado
  6. Sistema armazena o comunicado localmente
  7. Sistema sincroniza os dados quando houver conexão com a internet

UC05 - Visualizar Notas e Frequência

Ator Principal: Aluno, Responsável

Descrição: Este caso de uso permite aos alunos e responsáveis visualizarem notas e frequência.

Fluxo Principal:

  1. Usuário acessa a opção "Notas e Frequência"
  2. Sistema exibe as disciplinas (para alunos) ou lista de alunos (para responsáveis)
  3. Usuário seleciona a disciplina ou aluno desejado
  4. Sistema exibe as notas e frequência correspondentes

UC06 - Consultar Calendário Escolar

Ator Principal: Todos os usuários

Descrição: Este caso de uso permite aos usuários consultarem o calendário escolar.

Fluxo Principal:

  1. Usuário acessa a opção "Calendário Escolar"
  2. Sistema exibe o calendário do mês atual
  3. Usuário pode navegar entre os meses
  4. Usuário seleciona uma data para ver detalhes dos eventos

UC07 - Registrar Ocorrências Disciplinares

Ator Principal: Professor, Administrador

Descrição: Este caso de uso permite registrar ocorrências disciplinares dos alunos.

Fluxo Principal:

  1. Usuário acessa a opção "Ocorrências Disciplinares"
  2. Usuário seleciona "Nova Ocorrência"
  3. Usuário seleciona o aluno
  4. Usuário preenche os detalhes da ocorrência
  5. Usuário registra a ocorrência
  6. Sistema notifica o responsável
