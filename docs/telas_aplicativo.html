Telas do Aplicativo - EscolApp

Introdução

Esta página apresenta as principais telas implementadas no protótipo do EscolApp, um sistema simplificado de gestão escolar para dispositivos móveis, desenvolvido para atender às necessidades da Escola Municipal Professor Paulo Freire.

As telas foram implementadas utilizando Python com o framework Kivy/KivyMD, seguindo os protótipos de interface e mantendo o foco em simplicidade, usabilidade e desempenho em dispositivos com recursos limitados.

Descrição

A tela de login é a porta de entrada do aplicativo, permitindo que os usuários se autentiquem com suas credenciais. Esta tela foi implementada com foco na simplicidade e facilidade de uso.

Funcionalidades Implementadas

  • Autenticação de usuários com diferentes perfis (Administrador, Professor, Aluno, Responsável)
  • Validação de credenciais com feedback visual para erros
  • Armazenamento seguro de credenciais para login automático em acessos futuros
  • Opção de recuperação de senha (envia instruções para o e-mail cadastrado)
  • Funcionamento offline com validação local de credenciais

Implementação Técnica

A tela foi implementada utilizando componentes MDTextField do KivyMD para os campos de entrada, com validação em tempo real. A autenticação é realizada contra o banco de dados SQLite local, com hash de senhas para segurança.

Descrição

O menu principal é o hub central do aplicativo, oferecendo acesso a todas as funcionalidades disponíveis para o perfil do usuário. Esta tela se adapta automaticamente ao perfil do usuário logado, mostrando apenas as opções pertinentes.

Funcionalidades Implementadas

  • Menu dinâmico que se adapta ao perfil do usuário (Administrador, Professor, Aluno, Responsável)
  • Indicadores visuais para notificações não lidas
  • Identificação clara do usuário logado e seu perfil
  • Navegação intuitiva para as diferentes funcionalidades do sistema
  • Opção de logout seguro

Implementação Técnica

A tela foi implementada utilizando MDList e MDListItem do KivyMD para criar o menu interativo. A adaptação ao perfil do usuário é feita dinamicamente no momento do carregamento da tela, com base nas permissões armazenadas no banco de dados local.

Descrição

A tela de registro de notas permite aos professores inserir e editar as notas dos alunos de uma turma específica. Esta funcionalidade é essencial para o acompanhamento do desempenho acadêmico.

Funcionalidades Implementadas

  • Seleção de turma, disciplina e bimestre
  • Visualização da lista completa de alunos da turma selecionada
  • Entrada de notas com validação (valores de 0 a 10 com até uma casa decimal)
  • Salvamento automático local para evitar perda de dados
  • Indicador de status de sincronização
  • Histórico de alterações para rastreabilidade

Implementação Técnica

A tela foi implementada utilizando MDDataTable do KivyMD para exibir a lista de alunos e suas notas. Os dados são carregados do banco de dados SQLite local e as alterações são salvas automaticamente. A sincronização é realizada quando há conexão com a internet.

Descrição

A tela de comunicados permite visualizar mensagens enviadas pela escola, professores e outros usuários, além de permitir o envio de novos comunicados (dependendo do perfil do usuário). Esta funcionalidade é essencial para a comunicação entre escola e famílias.

Funcionalidades Implementadas

  • Visualização de comunicados recebidos, ordenados por data
  • Criação de novos comunicados (para perfis autorizados)
  • Categorização de comunicados (geral, específico para turma, individual)
  • Indicadores de status de leitura
  • Armazenamento local para acesso offline
  • Sincronização quando há conexão com a internet
  • Notificações para novos comunicados

Implementação Técnica

A tela foi implementada utilizando MDCard do KivyMD para exibir cada comunicado. Os dados são carregados do banco de dados SQLite local com paginação para otimizar o desempenho. A criação de novos comunicados é feita através de um diálogo modal com campos para título, conteúdo e seleção de destinatários.

Considerações Técnicas

As telas apresentadas foram implementadas utilizando Python com o framework Kivy/KivyMD, seguindo as seguintes diretrizes técnicas:

  • Arquitetura MVC: Separação clara entre modelo (dados), visão (interface) e controlador (lógica)
  • Banco de Dados Local: SQLite para armazenamento local de dados
  • Funcionamento Offline: Todas as funcionalidades principais funcionam sem necessidade de conexão com a internet
  • Sincronização: Mecanismo de sincronização quando há conexão disponível
  • Otimização de Desempenho: Carregamento progressivo de dados, uso eficiente de memória
  • Responsividade: Adaptação a diferentes tamanhos de tela e orientações
  • Acessibilidade: Elementos de tamanho adequado, contraste suficiente, suporte a recursos de acessibilidade do sistema

Próximos Passos

O protótipo implementado demonstra as funcionalidades principais do EscolApp, mas ainda há espaço para melhorias e expansões:

  • Implementação completa de todas as funcionalidades planejadas
  • Refinamento da interface com base em feedback de usuários reais
  • Melhorias de desempenho para dispositivos com recursos muito limitados
  • Expansão das funcionalidades de sincronização
  • Implementação de recursos adicionais de acessibilidade
  • Testes extensivos em diferentes dispositivos e condições de uso
