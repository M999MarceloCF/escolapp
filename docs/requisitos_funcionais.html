Requisitos Funcionais - EscolApp

Este documento apresenta os requisitos funcionais do sistema EscolApp, um aplicativo mobile simplificado para gestão escolar, desenvolvido para a Escola Municipal Professor Paulo Freire.

RF01 - Gestão de Usuários

Descrição: O sistema deve permitir o cadastro, edição, consulta e exclusão de usuários com diferentes perfis de acesso.

  • RF01.1: Cadastrar usuários com os seguintes perfis: direção, coordenação, professor, secretaria, pais/responsáveis.
  • RF01.2: Permitir a edição de dados cadastrais dos usuários.
  • RF01.3: Permitir a consulta de usuários cadastrados.
  • RF01.4: Permitir a inativação de usuários (exclusão lógica).
  • RF01.5: Controlar o acesso às funcionalidades de acordo com o perfil do usuário.

RF02 - Autenticação

Descrição: O sistema deve controlar o acesso através de login e senha.

  • RF02.1: Permitir login com nome de usuário e senha.
  • RF02.2: Permitir recuperação de senha através de e-mail cadastrado.
  • RF02.3: Encerrar sessão após período de inatividade.
  • RF02.4: Permitir que o usuário altere sua própria senha.

RF03 - Gestão de Turmas

Descrição: O sistema deve permitir o cadastro e gerenciamento de turmas.

  • RF03.1: Cadastrar turmas com informações como nome, ano letivo e período.
  • RF03.2: Associar alunos às turmas.
  • RF03.3: Associar professores e disciplinas às turmas.
  • RF03.4: Permitir a consulta de turmas e seus integrantes.

RF04 - Registro de Notas

Descrição: O sistema deve permitir o registro e consulta de notas dos alunos.

  • RF04.1: Permitir que professores registrem notas por aluno, disciplina e bimestre.
  • RF04.2: Calcular automaticamente a média bimestral e anual.
  • RF04.3: Permitir a consulta de notas pelos alunos e pais/responsáveis.
  • RF04.4: Permitir a geração de boletins por aluno.

RF05 - Registro de Frequência

Descrição: O sistema deve permitir o registro e consulta de frequência dos alunos.

  • RF05.1: Permitir que professores registrem presença/ausência por aluno, disciplina e data.
  • RF05.2: Calcular automaticamente o percentual de frequência.
  • RF05.3: Alertar sobre alunos com frequência abaixo do mínimo exigido (75%).
  • RF05.4: Permitir a consulta de frequência pelos alunos e pais/responsáveis.

RF06 - Comunicados

Descrição: O sistema deve permitir o envio e recebimento de comunicados entre escola e famílias.

  • RF06.1: Permitir que a direção, coordenação e secretaria criem comunicados.
  • RF06.2: Permitir a aprovação de comunicados pela direção antes do envio.
  • RF06.3: Enviar comunicados para turmas específicas ou para toda a escola.
  • RF06.4: Permitir a visualização de comunicados pelos pais/responsáveis.
  • RF06.5: Registrar a leitura dos comunicados pelos destinatários.

RF07 - Calendário Escolar

Descrição: O sistema deve disponibilizar o calendário escolar com eventos e datas importantes.

  • RF07.1: Permitir que a direção e coordenação cadastrem eventos no calendário.
  • RF07.2: Categorizar eventos (feriados, reuniões, provas, etc.).
  • RF07.3: Permitir a visualização do calendário por todos os usuários.
  • RF07.4: Enviar notificações sobre eventos próximos.

RF08 - Ocorrências Disciplinares

Descrição: O sistema deve permitir o registro e acompanhamento de ocorrências disciplinares.

  • RF08.1: Permitir que professores e coordenação registrem ocorrências por aluno.
  • RF08.2: Classificar ocorrências por tipo e gravidade.
  • RF08.3: Notificar pais/responsáveis sobre ocorrências registradas.
  • RF08.4: Permitir a consulta do histórico de ocorrências por aluno.

RF09 - Relatórios

Descrição: O sistema deve gerar relatórios para acompanhamento pedagógico e administrativo.

  • RF09.1: Gerar relatório de desempenho por aluno, turma e disciplina.
  • RF09.2: Gerar relatório de frequência por aluno, turma e disciplina.
  • RF09.3: Gerar relatório de ocorrências disciplinares.
  • RF09.4: Permitir a exportação de relatórios em formato PDF.

RF10 - Funcionamento Offline

Descrição: O sistema deve permitir o funcionamento básico mesmo sem conexão com a internet.

  • RF10.1: Armazenar dados localmente no dispositivo.
  • RF10.2: Permitir o registro de notas e frequência offline.
  • RF10.3: Sincronizar dados automaticamente quando houver conexão.
  • RF10.4: Indicar visualmente o status de sincronização dos dados.
