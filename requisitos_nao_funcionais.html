Requisitos Não Funcionais - EscolApp

Este documento apresenta os requisitos não funcionais do sistema EscolApp, um aplicativo mobile simplificado para gestão escolar, desenvolvido para a Escola Municipal Professor Paulo Freire.

RNF01 - Usabilidade

Descrição: O sistema deve ser intuitivo e de fácil uso, mesmo para usuários com pouca experiência tecnológica.

  • RNF01.1: A interface deve ser simples e intuitiva, com elementos visuais claros e consistentes.
  • RNF01.2: O sistema deve exigir no máximo 3 cliques para acessar qualquer funcionalidade.
  • RNF01.3: O sistema deve fornecer feedback visual para todas as ações do usuário.
  • RNF01.4: O sistema deve incluir ajuda contextual para as principais funcionalidades.
  • RNF01.5: O sistema deve ser utilizável por pessoas com diferentes níveis de familiaridade com tecnologia.

RNF02 - Desempenho

Descrição: O sistema deve apresentar tempo de resposta adequado, mesmo em dispositivos com configurações modestas.

  • RNF02.1: O tempo de resposta para operações básicas deve ser inferior a 3 segundos.
  • RNF02.2: O sistema deve suportar até 500 usuários simultâneos sem degradação significativa de desempenho.
  • RNF02.3: O consumo de memória RAM não deve exceder 100MB durante o uso normal.
  • RNF02.4: O sistema deve funcionar adequadamente em dispositivos com processadores de 1.0 GHz e 1GB de RAM.

RNF03 - Compatibilidade

Descrição: O sistema deve ser compatível com diferentes versões do sistema operacional Android.

  • RNF03.1: O sistema deve ser compatível com Android 6.0 (Marshmallow) ou superior.
  • RNF03.2: O sistema deve se adaptar a diferentes tamanhos e resoluções de tela.
  • RNF03.3: O sistema deve manter a consistência visual e funcional em diferentes dispositivos.

RNF04 - Disponibilidade

Descrição: O sistema deve estar disponível mesmo sem conexão constante com a internet.

  • RNF04.1: As funcionalidades básicas devem estar disponíveis offline.
  • RNF04.2: O sistema deve sincronizar automaticamente os dados quando houver conexão com a internet.
  • RNF04.3: O sistema deve indicar claramente o status de sincronização dos dados.
  • RNF04.4: O sistema deve resolver conflitos de sincronização de forma automática ou assistida.

RNF05 - Segurança

Descrição: O sistema deve garantir a segurança e privacidade dos dados.

  • RNF05.1: As senhas devem ser armazenadas de forma criptografada.
  • RNF05.2: O acesso às funcionalidades deve ser controlado de acordo com o perfil do usuário.
  • RNF05.3: A comunicação com o servidor deve ser criptografada (HTTPS).
  • RNF05.4: O sistema deve encerrar a sessão após 30 minutos de inatividade.
  • RNF05.5: O sistema deve registrar logs de acesso e operações críticas.

RNF06 - Escalabilidade

Descrição: O sistema deve suportar o crescimento da base de usuários e dados.

  • RNF06.1: O sistema deve suportar até 1000 alunos cadastrados.
  • RNF06.2: O sistema deve permitir o armazenamento de dados históricos por pelo menos 5 anos.
  • RNF06.3: O sistema deve ser modular, permitindo a adição de novas funcionalidades sem impactar as existentes.

RNF07 - Manutenibilidade

Descrição: O sistema deve ser fácil de manter e atualizar.

  • RNF07.1: O código-fonte deve seguir padrões de codificação e boas práticas.
  • RNF07.2: O sistema deve ser modular, com baixo acoplamento entre componentes.
  • RNF07.3: O sistema deve incluir documentação técnica completa.
  • RNF07.4: As atualizações devem ser instaláveis sem perda de dados.

RNF08 - Portabilidade

Descrição: O sistema deve ser portável para diferentes ambientes.

  • RNF08.1: O sistema deve ser desenvolvido com tecnologias multiplataforma.
  • RNF08.2: O banco de dados deve ser independente de plataforma.
  • RNF08.3: O sistema deve permitir a exportação de dados em formatos padrão.

RNF09 - Tamanho e Instalação

Descrição: O sistema deve ser leve e de fácil instalação.

  • RNF09.1: O tamanho do aplicativo instalado não deve exceder 30MB.
  • RNF09.2: O processo de instalação deve ser simples e guiado.
  • RNF09.3: O sistema deve requerer permissões mínimas necessárias para seu funcionamento.

RNF10 - Acessibilidade

Descrição: O sistema deve ser acessível para usuários com diferentes necessidades.

  • RNF10.1: O sistema deve seguir diretrizes básicas de acessibilidade.
  • RNF10.2: O sistema deve permitir o ajuste do tamanho da fonte.
  • RNF10.3: O sistema deve utilizar cores com contraste adequado.
  • RNF10.4: O sistema deve ser compatível com recursos de acessibilidade do Android.
